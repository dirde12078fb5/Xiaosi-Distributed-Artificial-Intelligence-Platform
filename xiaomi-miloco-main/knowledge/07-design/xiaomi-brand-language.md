# 小米品牌设计语言

> 设计 SSOT — 小米 PPT/品牌设计系统全量规范
> 本文档是小米品牌的"原汁原味"全量规范。Family UI 是它的 Web 子集,有取舍——见末尾"与 Family UI 的对应关系"。

---

## 1. 视觉主题与氛围

小米的设计建立在**高对比度、数据驱动**的视觉基础上:**纯白画布(`#FFFFFF`)+ 小米橙(`#FF6900`)+ 黑(`#1A1A1A`)** 三色体系。

视觉隐喻:**实验室仪器面板**——白色背板承载黑色刻度与读数,橙色用于指示关键状态与警戒线。

设计哲学:**让信息说话**——

1. 橙色品牌色一致性应用(表头 / 进度条 / 编号 / CTA / 装饰线)
2. 多样化版式布局适配不同内容类型(10 种核心版式)
3. 清晰的视觉层级(尺寸 / 颜色 / 留白三维度)
4. 极简几何装饰(左侧 8px 竖线 + 底部 16-24px 橙色条带,仅此两类)

与其他设计语言的差异:

- ≠ Apple 大留白美学 → 小米偏"高密度技术信息可读性"
- ≠ 传统企业 PPT → 不用立体透视、3D 图表、装饰云朵
- ≠ 咨询公司模板 → 不用 Alpha/Beta 等抽象符号

## 2. 色彩体系

### 2.1 品牌色

| 色     | 值        | 用途                                                                  |
| ------ | --------- | --------------------------------------------------------------------- |
| 小米橙 | `#FF6900` | **唯一**品牌强调色——表头、进度条、数字强调、底部横幅、Checkmark、编号 |
| 深橙   | `#E65100` | Hover / 次级强调 / 渐变暗端                                           |

### 2.2 文字色阶

| 色     | 值        | 用途                             |
| ------ | --------- | -------------------------------- |
| 主文字 | `#1A1A1A` | 标题、正文、表格内容             |
| 次要   | `#666666` | 副标题、描述、辅助说明           |
| 三级   | `#999999` | 标签、注释、提示(不用于核心信息) |
| 反色   | `#FFFFFF` | 橙色/黑色背景上的文字            |

> **使用原则**:同一页面文字色阶不超过 3 档,避免"灰色大家族"。

### 2.3 中性色阶

`#FFFFFF` → `#FAFAFA` → `#F5F5F5` → `#E8E8E8` → `#D5D5D5` → `#999999` → `#666666` → `#333333` → `#000000`

| 色        | 用途                     |
| --------- | ------------------------ |
| `#FAFAFA` | 卡片填充(与背景轻微区分) |
| `#F5F5F5` | 表格斑马纹偶数行         |
| `#E8E8E8` | 水平分隔线(1px 默认)     |
| `#D5D5D5` | 纵向分隔线(1px,左右分栏) |

### 2.4 语义色

| 色        | 值             | 用途               |
| --------- | -------------- | ------------------ |
| 成功/增长 | `#52C41A` (绿) | KPI 正向变化       |
| 警告      | `#FAAD14` (黄) | 待处理、延期、提醒 |
| 错误/下降 | `#F5222D` (红) | KPI 负向变化、失败 |
| 信息      | `#1890FF` (蓝) | 中性提示、外部链接 |

> 语义色仅用于状态表达,不参与品牌表达。一个橙色按钮旁不应同时出现绿色按钮——会让观众困惑优先级。

### 2.5 数据可视化调色板

```
主色 #FF6900    辅色 #FFA040    第三 #FFD699
对比 #1890FF    中性 #999999
```

色序使用:多系列图表按 `#FF6900 → #FFA040 → #FFD699 → #1890FF → #999999` 顺序分配,前三色饱和度递减形成层次。

### 2.6 配色禁区

- ❌ **粉色**(`#FF4D6D`、`#F94B6D`)—— 与小米橙冲突,显廉价
- ❌ **紫色 / 品红** —— 品牌外
- ❌ **纯红 `#FF0000`** —— 刺眼,用 `#FF6900` 替代
- ❌ **多色渐变** —— 破坏"数据优先"哲学
- ❌ **超过 2 种饱和度相同的暖色同屏**

## 3. 字体规则

### 3.1 字体家族

| 用途     | family                                                                    |
| -------- | ------------------------------------------------------------------------- |
| 标题     | `MiSans, PingFang SC, Microsoft YaHei, sans-serif`                        |
| 正文     | `PingFang SC, MiSans, Microsoft YaHei, Helvetica Neue, Arial, sans-serif` |
| **数字** | `DIN Alternate, MiSans, Arial, sans-serif`(粗体 700-900)                  |
| 等宽     | `SF Mono, Consolas, Courier New, monospace`                               |

### 3.2 字号层级(PPT @144DPI)

| 层级       | px     | pt      | 字重    | 行高 | 用途                       |
| ---------- | ------ | ------- | ------- | ---- | -------------------------- |
| Display    | 96-120 | 48-60   | 800-900 | 1.0  | Hero 数字:2.5x, 60B, 99.9% |
| H1         | 48-64  | 24-32   | 700     | 1.1  | 页面主标题                 |
| H2         | 36-40  | 18-20   | 700     | 1.2  | 数据标签                   |
| H3         | 24-32  | 12-16   | 600-700 | 1.3  | 卡片标题、表格列标题       |
| Body       | 16-18  | 8-9     | 400     | 1.6  | 正文、表格内容             |
| Body Small | 14     | 7       | 400     | 1.5  | 辅助说明、标签             |
| Caption    | 12     | 6       | 400     | 1.4  | 注释、页码                 |
| Overline   | 11-13  | 5.5-6.5 | 600     | 1.3  | 章节编号 01-05             |

> **1 pt = 2 px @ 144 DPI 约定**。

### 3.3 字重使用规则

- **400 常规** —— 正文、表格、辅助说明
- **500-600 中粗** —— 副标题、小标题、强调短语
- **700 粗** —— 主标题、表格列标题、按钮文字
- **800-900 重** —— 仅 Hero 数字 / Display,营造数据冲击力

### 3.4 排版搭配

1. **标题粗 + 正文细** —— 永远不要让正文字重追上标题
2. **数字超大 + 标签适中** —— KPI 数字要比标签字号大 2-3 倍
3. **中文不做斜体** —— 中文字体斜体只破坏可读性
4. **标题不做下划线**

### 3.5 中英混排

- 中英文之间**不插空格**(正文)
- 代码 / 数字与中文间保留半角空格("响应时间 42ms" 不是 "响应时间42ms")
- 英文品牌名保持原大小写(Mi、IoT、MiOT)

## 4. 间距系统

### 4.1 基础

- **基准单位**:8px
- **间距序列**:`4 / 8 / 12 / 16 / 24 / 32 / 48 / 64 / 80 / 96 / 128`px
- **严格对齐**:任何坐标 / 尺寸 / 间距必须是 4px 整数倍;> 24px 必须是 8px 整数倍

### 4.2 页面边距(1920×1080 画布)

| 方向 | px     | 备注                   |
| ---- | ------ | ---------------------- |
| 上   | 64-80  | Hero 页可压缩至 48     |
| 左右 | 80-100 | 内容版心宽度 1720-1760 |
| 下   | 48-64  | 若有底部橙色条带则另加 |

### 4.3 元素间距

| 关系               | 间距               |
| ------------------ | ------------------ |
| 卡片内边距         | 32-40px            |
| 卡片之间 gap       | 24-32px            |
| 表格单元格 padding | 12×16px(行高 56px) |
| 段落间距           | 24-32px            |
| 同组元素           | 16-24px            |
| 分组间距           | 48-64px            |

### 4.4 留白哲学

留白不是装饰,**是信息架构的分隔符**:

- 同类信息之间用小留白(16-24px)
- 不同类信息之间用中等留白(48-64px)
- 页面大模块之间用大留白(80-120px)

## 5. 形状与装饰

### 5.1 圆角规则

| 元素     | 圆角         | 原因                 |
| -------- | ------------ | -------------------- |
| 按钮     | 8px          | 柔和但不过度         |
| 卡片     | 16px         | 明确"容器感"         |
| 卡片紧凑 | 12px         | 密集列表             |
| 进度条   | 999px (pill) | 胶囊形强化"进度"隐喻 |
| 编号徽章 | 50% (圆)     | 01-05 编号圆形标记   |
| 表格     | 0px (直角)   | 表格需要网格感       |
| 输入框   | 4-8px        | 与按钮保持一致       |

### 5.2 装饰元素(仅两类)

1. **左侧橙色装饰线**——8px 宽 `#FF6900`,延伸页面 60-80% 高度,作为贯穿品牌识别(content 类页可选)
2. **底部橙色条带**——16-24px 高 `#FF6900`,用于视觉收尾,可含白色文字 CTA

> 除此之外:**不**使用装饰图案(云朵、几何块、抽象线条)、**不**使用流体形状、**不**使用光效。

### 5.3 阴影层级

| 层级   | css                           | 场景             |
| ------ | ----------------------------- | ---------------- |
| flat   | `none`                        | 表格、列表(默认) |
| low    | `0 2px 4px rgba(0,0,0,0.04)`  | 卡片、内容面板   |
| medium | `0 4px 12px rgba(0,0,0,0.08)` | 悬浮卡片、下拉   |
| high   | `0 8px 24px rgba(0,0,0,0.12)` | 模态框、顶层弹窗 |

> 阴影要"几乎察觉不到"才算对。

## 6. 10 种核心版式(PPT 场景)

| 编号 | 名称         | 结构                                         | 典型场景           |
| ---- | ------------ | -------------------------------------------- | ------------------ |
| L01  | 封面页       | 左侧大号标题 + Logo 右上 + 橙色装饰线        | 全文开篇           |
| L02  | 章节页       | 居中大号橙色编号 + 章节标题列表              | 多章分段           |
| L03  | Hero 数据页  | 左侧超大橙色数字 + 右侧要点列表 + 8px 装饰线 | 核心 KPI 单点强调  |
| L04  | 全宽表格页   | 橙色表头 + 斑马纹                            | 多指标对比、明细   |
| L05  | 三栏卡片     | 三等宽卡片 + 橙色编号 + 底部条带             | 流程步骤、并列方案 |
| L06  | 编号列表     | 左侧橙色编号 + 右侧标题正文                  | 要点清单、步骤分解 |
| L07  | 四栏指标     | 橙色大百分比 + 说明 + 底部 CTA               | 季度指标           |
| L08  | 甘特图       | 时间轴 + 橙色进度条                          | 项目节奏、Roadmap  |
| L09  | 进度条列表   | 多胶囊进度条 + 右侧百分比                    | OKR 完成度         |
| L10  | 左右分栏指标 | 左大数字 + 右标题说明                        | 2-3 个并列 KPI     |

版式选择原则:

- 1 个核心数字 → L03
- 2-3 个并列数字 → L10
- 4 个指标 → L07
- 5+ 指标 → L09 / L04
- 步骤/流程 → L06
- 并列方案 → L05
- 时间维度 → L08
- 详细数据 → L04

## 7. 数据可视化

| 场景              | 推荐图表                     |
| ----------------- | ---------------------------- |
| 数值对比(< 10 项) | 柱状 / 条形                  |
| 趋势变化          | 折线                         |
| 占比(2-4 项)      | 环形(donut,**不用饼图**)     |
| 多维对比          | 雷达(慎用)                   |
| 流程/漏斗         | 漏斗 / 流程图                |
| 时间跨度          | 甘特(L08)                    |
| 相关性            | 散点                         |
| 进度              | **胶囊进度条**(不是圆环进度) |

数值展示:

- 整数:`2,381` 直接显示
- 小数:最多 2 位
- 百分比:带 % 不分开(`99.9%` 不是 `99.9 %`)
- 大数字:K/M/B 缩写(`1.2K` / `3.5M` / `60B`)
- delta:`+ / -` 前缀 + 颜色区分(`+2.3%` 绿,`-1.1%` 红)

禁忌:3D 柱、饼图 > 5 片、多色渐变柱、数据标签遮坐标轴。

## 8. 动效与交互

### 8.1 时长

| 时长      | 用途                             |
| --------- | -------------------------------- |
| 100ms     | 按钮按下、悬停微变化             |
| 200-250ms | 元素状态切换(hover、color)       |
| 300-400ms | 页面过渡、弹窗入场、抽屉滑出     |
| 600ms+    | **慎用**——仅刻意营造"仪式感"开场 |

### 8.2 缓动曲线

```
入场   cubic-bezier(0.0, 0.0, 0.2, 1)   ease-out  快进慢停
出场   cubic-bezier(0.4, 0.0, 1, 1)     ease-in   慢进快离
切换   cubic-bezier(0.4, 0.0, 0.2, 1)   ease-in-out
```

### 8.3 加载状态

- 进度条:0 向目标填充,300-500ms `ease-in-out`
- 数字计数:0 向目标滚动,400-600ms
- 骨架屏:浅灰 → 浅橙渐变循环,1200ms loop

> 禁忌:**不在 PPT 中用旋转加载圆环**(语境错位);Web 中可用。

## 9. 品牌语调(Brand Voice)

四组反义词定位:

| **是**     | **不是** |
| ---------- | -------- |
| 直接       | 含糊     |
| 数据化     | 故事化   |
| 陈述式     | 抒情式   |
| 工程师语感 | 营销语感 |

写作原则:

1. **数字优先于形容词** —— "快 2.3 倍"而不是"显著提升"
2. **短句优先于长句** —— 单句 ≤ 20 字
3. **陈述句优先于感叹句** —— 不用 `!`
4. **具体优先于抽象** —— "降低 38% 延迟"而不是"优化性能"
5. **可验证优先于宣传** —— "基于 10 万用户测试"不是"广受好评"

✅ 推荐文案:

- "响应时间 42ms,较上季度提升 38%"
- "支持 4800 万像素主摄 + 1200 万像素广角"
- "已部署至 18 国 / 67 城"

❌ 避免文案:

- "响应快如闪电,体验前所未有"
- "让梦想照进现实的旗舰之作"
- "遍布全球的智能生态"

## 10. 无障碍

| 维度                   | 标准                                                  |
| ---------------------- | ----------------------------------------------------- |
| 正文对比度             | ≥ 4.5:1(WCAG AA)                                      |
| 大字号(18pt+)          | ≥ 3:1                                                 |
| **橙色 #FF6900 on 白** | 对比度 3.05:1 → 仅可用于大字号和装饰,**不可用于正文** |
| 投屏正文最小           | 18pt(辅助 14pt)                                       |
| PDF 正文最小           | 16pt(辅助 12pt)                                       |

**颜色不可独行**(色盲友好):

- ❌ 仅红/绿表示"下降/上升"—— 同时加 `↓` / `↑` 或 `+` / `-`
- ❌ 仅橙色表示"点击这里"—— 加下划线或箭头
- ✅ 颜色 + 符号 / 颜色 + 文字 / 颜色 + 形状

## 11. 与 Family UI 的对应关系

Family UI(`web`)是小米品牌的 **Web 子集**,做了如下取舍:

### 保留

| 维度        | 完整继承                           |
| ----------- | ---------------------------------- |
| 视觉哲学    | 数据密度 > 装饰                    |
| 视觉隐喻    | 实验室仪器面板                     |
| 主品牌色    | 小米橙 + 白底黑字三色体系          |
| 字体策略    | 中文走系统 MiSans 栈 + 数字走 mono |
| 留白节奏    | 8px 基准 + 间距序列                |
| Brand Voice | 数据化 / 陈述式 / 工程师语感       |

### 调整

| 维度     | PPT 体系                        | Family UI 体系                 | 原因                                |
| -------- | ------------------------------- | ------------------------------ | ----------------------------------- |
| 主橙值   | `#FF6900`                       | `#FF6700`                      | Web token 库历史值,差 1° 色相       |
| 数字字体 | `DIN Alternate`(显眼粗号)       | `Geist Mono`                   | dev tool 风,屏幕渲染更稳            |
| 卡片圆角 | 16px (`rounded-2xl`)            | **12px (`rounded-xl`)**        | 16px 留给 modal/drawer,Web 卡片更紧 |
| 装饰元素 | 8px 橙色竖线 + 底部条带         | **不用**                       | 信息密度反而被装饰打断              |
| 阴影     | `0 2px 4px rgba(0,0,0,.04)` low | `0 1px 2px rgba(0,0,0,.04)` sm | Web 上更隐形                        |
| 状态表达 | 绿/红/黄实色 chip               | **5px 状态点 + 8% 光环**       | 减少色块对白底的破坏                |
| 字重     | 400 / 500 / 600 / 700 / 800     | **400 / 600 两档**             | 14px 上 500 跟 400 几乎无差         |

### 不引入(PPT 专属)

- 10 种核心版式(L01-L10)—— Web 不需要"汇报型 deck 流"
- DIN Alternate 数字字体 —— Web 用 Geist Mono 更稳
- 左侧橙色 8px 竖线 + 底部橙色条带装饰 —— 减弱"数据密度"
- 橙色实色表头 + 斑马纹 —— Web 表格用 hairline + hover 替代

> **核心:Family UI 把小米品牌从"PPT 工程化审美"翻译到"Web dev tool 控制台审美",保留品牌色 + 留白哲学 + 数据导向,放弃装饰元素与 PPT 专属版式。**
