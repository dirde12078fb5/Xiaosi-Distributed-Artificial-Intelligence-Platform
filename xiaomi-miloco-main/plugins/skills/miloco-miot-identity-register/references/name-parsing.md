# 姓名 / 家庭角色解析详细规则

> 本文件由 [SKILL.md](../SKILL.md) 第二步 2.2 引用。覆盖从用户一句话里抽取 name + role 两个字段的完整规则与边界。

用户在注册指令里**通常会同时提供姓名 + 家庭角色**, agent 要从一句话里抽出两个字段:

| 字段        | 来源        | 含义                                   |
|-------------|-------------|----------------------------------------|
| `name`      | 用户给的全名 / 真名 | 写入 `--member-name`, 唯一标识     |
| `role`      | 用户给的家庭角色 / 关系 | 写入 `--member-role`, 平时显示用 |

## 抽取规则

| 用户原话                    | name      | role     |
|-----------------------------|-----------|----------|
| "这是我爸爸李建国, 帮我注册" | 李建国    | 爸爸     |
| "登记一下张三"              | 张三      | (空)     |
| "这是小赵"                  | 小赵      | (空)     |
| "我们家保姆王阿姨"          | 王阿姨    | 保姆     |
| "这是我妈"                  | (空,追问) | 妈妈     |
| "我自己,叫小赵"             | 小赵      | (我)     |

## 判定逻辑

- 句中出现亲属关系词("爸爸 / 妈妈 / 哥 / 姐 / 儿子 / 女儿 / 爷爷 / 奶奶 / 保姆 / 阿姨 ..." 等) → 作 role
- 剩下的人名(2~4 个汉字 + 不是亲属词) → 作 name
- **只有亲属角色没有真名**(如"这是我妈") → 必须追问"她叫什么名字?"
- **只有真名没有家庭角色**(如"登记张三") → 省略 `--member-role`, role 留空即可; name 照常写入, **不会**被任何值兜底顶替

## 调用模板

```bash
miloco-cli identity register commit \
    --pending-id <id> --indices 0,3,5 \
    --member-name "李建国" --member-role "爸爸" --pretty
```

```bash
miloco-cli identity register from-cluster \
    --cluster-id <id> \
    --name "李建国" --role "爸爸" --pretty
```

> ⚠️ CLI 里 `register commit` 用的是 `--member-name` / `--member-role`, 而 `register from-cluster` 用的是 `--name` / `--role`, **前缀不一致是 CLI 历史遗留**, 以本文件命令为准, 不要凭直觉对调。
