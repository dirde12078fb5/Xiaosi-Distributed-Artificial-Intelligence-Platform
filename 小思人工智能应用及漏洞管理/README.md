## 🌟 项目简介

**小思人工智能应用及漏洞管理** 是小思分布式人工智能平台的核心安全管理模块，基于 Python `tkinter` 开发的现代化桌面应用程序。它集成了AI应用管理与安全漏洞扫描两大核心功能，为AI系统提供全方位的安全防护与管理能力。

本模块采用多线程并发架构，确保在进行深度安全扫描时，UI界面依然保持流畅响应。支持系统漏洞检测、网络端口扫描、AI模型安全评估等多种安全检测能力。

## 📖 使用说明

```bash
# Windows
python 小思人工智能应用及漏洞管理.py

# Linux / macOS
python3 小思人工智能应用及漏洞管理.py
```

启动成功后，你将看到主界面，包含顶部标题栏、中间标签页区域和底部状态栏。

### 📊 仪表盘使用

仪表盘是程序的首页，提供全局概览：

#### 统计卡片
- **AI模型数量**：当前已注册的AI模型总数
- **运行中应用**：状态为"运行中"的模型数量
- **已发现漏洞**：最近一次扫描发现的漏洞数量
- **安全评分**：根据漏洞数量计算的综合安全评分（0-100）

#### 系统信息卡片
显示当前系统的硬件和软件信息：
- 操作系统及版本
- 处理器型号
- 主机名
- 内存总量
- CPU核心数

#### 最近活动卡片
实时记录程序的所有操作，包括：
- 程序启动
- 模型添加/删除
- 扫描开始/结束
- 报告导出等

### 🤖 AI应用管理

#### 添加AI模型

1. 切换到 **"AI应用管理"** 标签页
2. 点击 **"添加模型"** 按钮
3. 在弹出的对话框中填写：
   - **模型名称**：（必填）模型的名称标识
   - **模型类型**：下拉选择（LLM、CV、语音、多模态、其他）
   - **版本**：模型版本号（可选，默认1.0）
   - **模型路径**：模型文件所在目录（可点击"浏览"选择）
4. 点击 **"确定"** 完成添加

#### 刷新模型状态

1. 点击工具栏中的 **"刷新状态"** 按钮
2. 程序将自动检查每个模型的路径是否存在
3. 更新模型状态为：运行中 / 已停止 / 未找到

#### 移除模型

1. 在模型列表中选中要移除的模型（支持多选）
2. 点击 **"移除选中"** 按钮
3. 在确认对话框中点击 **"是"**

> **提示**：模型配置会自动保存到 `ai_models_config.json` 文件中，下次启动时自动加载。

---

### 🔍 系统漏洞扫描

#### 开始扫描

1. 切换到 **"系统漏洞扫描"** 标签页
2. 选择扫描类型：
   - **快速扫描**：5项基础检测，速度最快
   - **完整扫描**：10项标准检测，推荐日常使用
   - **深度扫描**：18项全面检测，最全面但耗时较长
3. 点击 **"开始扫描"** 按钮
4. 扫描过程中可以查看：
   - 当前扫描项进度
   - 进度条显示
   - 实时发现的漏洞列表

#### 停止扫描

- 扫描过程中点击 **"停止扫描"** 按钮可随时中断扫描

#### 查看结果

扫描结果以表格形式展示，包含以下列：
- **严重程度**：严重/高危/中危/低危（颜色区分）
- **漏洞名称**：检测项名称
- **类别**：系统/安全/服务/网络/应用等
- **描述**：漏洞详细说明
- **状态**：待修复

#### 导出报告

1. 点击 **"导出报告"** 按钮
2. 选择保存位置和文件格式（.txt 或 .json）
3. 点击保存

**TXT格式示例**：
```
============================================================
        小思人工智能应用及漏洞管理 - 漏洞扫描报告
============================================================

扫描时间: 2026-06-30 12:00:00
漏洞总数: 3

------------------------------------------------------------

[1] 防火墙状态
    严重程度: high
    类别: 安全
    描述: 防火墙未完全启用或配置不当
    状态: 待修复

...
============================================================
```

### 🌐 网络端口扫描

#### 配置扫描参数

1. 切换到 **"网络端口扫描"** 标签页
2. 设置目标地址：默认 `127.0.0.1`（本机）
3. 设置端口范围：
   - 起始端口：默认 1
   - 结束端口：默认 1024
   - 有效范围：1 - 65535
4. 选择扫描速度：
   - **慢速**：准确率高，速度慢
   - **正常**：平衡模式（推荐）
   - **快速**：速度快，可能漏报

#### 开始扫描

1. 点击 **"开始扫描"** 按钮
2. 实时查看：
   - 当前扫描端口
   - 已发现开放端口数量
   - 进度条

#### 查看结果

扫描结果表格包含：
- **端口**：端口号
- **服务**：识别出的服务名称（如HTTP、SSH等）
- **状态**：开放/关闭
- **协议**：TCP/UDP

> **常见端口说明**：
> - 21: FTP 文件传输
> - 22: SSH 远程登录
> - 80: HTTP 网页服务
> - 443: HTTPS 加密网页
> - 3306: MySQL 数据库
> - 3389: RDP 远程桌面
> - 6379: Redis 缓存

### 🛡️ AI模型安全检测

#### 选择检测模型

1. 切换到 **"AI模型安全"** 标签页
2. 在"选择模型"下拉框中选择要检测的AI模型
   > 模型列表来自"AI应用管理"中已添加的模型

#### 选择检测项目

勾选需要检测的项目（默认全选）：
- ✅ 对抗性攻击检测
- ✅ 数据泄露风险
- ✅ 模型完整性校验
- ✅ 隐私合规检测

#### 开始检测

1. 点击 **"开始检测"** 按钮
2. 等待检测完成
3. 查看安全评分和检测结果

#### 安全评分说明

| 分数区间 | 等级 | 说明 |
|---------|------|------|
| 80-100 | 良好 | 安全状况良好 |
| 60-79 | 中等 | 存在中等风险，建议修复 |
| 0-59 | 危险 | 存在高风险，需立即处理 |

#### 检测结果说明

每项检测包含：
- **检测项**：检测名称
- **风险等级**：高/中/低
- **检测结果**：检测说明
- **建议**：修复或改进建议

### v1.0.0 (2026-06-30)

✨ **新功能**
- 首次发布小思人工智能应用及漏洞管理模块
- 仪表盘：统计概览、系统信息、活动日志
- AI应用管理：模型增删改查、状态监控
- 系统漏洞扫描：三种扫描模式、18+检测项
- 网络端口扫描：自定义范围、服务识别
- AI模型安全：四项安全检测、评分系统

🎨 **界面**
- 现代化卡片式UI设计
- 蓝色主题，与小思平台风格统一
- 支持窗口大小调整

🔧 **技术**
- 基于 Python tkinter + ttk 开发
- 多线程架构，扫描不卡UI
- JSON配置持久化
