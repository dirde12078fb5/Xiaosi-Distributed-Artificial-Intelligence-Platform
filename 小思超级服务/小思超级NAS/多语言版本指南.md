# 小思超级NAS - 多语言版本详细指南

## 📖 概述

本文档详细介绍小思超级NAS的6种语言实现，帮助您选择最适合的版本。

## 🔄 版本对比

| 版本 | 语言 | 性能 | 易用性 | 依赖 | 适用场景 |
|------|------|------|--------|------|----------|
| Python | Python 3 | ⭐⭐⭐ | ⭐⭐⭐⭐⭐ | Flask | 快速部署、原型开发 |
| Node.js | JavaScript | ⭐⭐⭐⭐ | ⭐⭐⭐⭐ | Express | Web开发、实时应用 |
| Java | Java 11+ | ⭐⭐⭐⭐ | ⭐⭐⭐ | 无额外依赖 | 企业应用、大型系统 |
| PHP | PHP 7.4+ | ⭐⭐⭐ | ⭐⭐⭐⭐⭐ | 无 | Web托管、共享主机 |
| C | C | ⭐⭐⭐⭐⭐ | ⭐⭐ | 无 | 高性能、嵌入式 |
| Shell | Bash | ⭐⭐⭐ | ⭐⭐⭐ | Python | Linux服务器 |

## 📁 统一的文件夹结构

每个语言版本都遵循统一的文件夹结构：

```
语言版本/
├── src/              # 源代码文件
├── config/           # 配置文件
│   ├── .env          # 环境变量配置
│   ├── config.xxx    # 语言特定配置
│   └── package.json  # Node.js依赖配置
└── scripts/          # 脚本文件
    ├── 启动.bat      # Windows启动脚本
    └── 编译.bat      # 编译脚本（仅C版本）
```

## 🚀 各版本详细说明

### 1. Python版本

**路径**: `01_python_version/`

**特点**:
- 易于部署，无需编译
- 自动安装依赖
- 跨平台支持

**启动方式**:
```bash
cd 01_python_version
scripts\启动.bat  # Windows
```

**配置文件**:
- `config/.env` - 服务器端口、JWT密钥等
- `config/requirements.txt` - Python依赖包

**依赖**:
- Flask
- Flask-CORS
- python-dotenv
- bcrypt
- PyJWT

---

### 2. Node.js版本

**路径**: `02_nodejs_version/`

**特点**:
- 高性能异步I/O
- 丰富的npm生态系统
- 适合Web应用开发

**启动方式**:
```bash
cd 02_nodejs_version
scripts\启动.bat  # Windows
```

**配置文件**:
- `config/.env` - 环境变量配置
- `config/package.json` - npm依赖和脚本

**依赖**:
- express
- cors
- jsonwebtoken
- bcryptjs

**安装依赖**:
```bash
cd config
npm install
```

---

### 3. Java版本

**路径**: `03_java_version/`

**特点**:
- 企业级可靠性
- 无需额外依赖（JDK自带HttpServer）
- 强类型语言

**启动方式**:
```bash
cd 03_java_version
scripts\启动.bat  # Windows
```

**配置文件**:
- `config/config.properties` - 服务器和用户配置

**要求**:
- JDK 11 或更高版本
- 自动编译，无需手动javac

---

### 4. PHP版本

**路径**: `04_php_version/`

**特点**:
- 广泛兼容
- 易于部署到Web托管
- 无需编译

**启动方式**:
```bash
cd 04_php_version
scripts\启动.bat  # Windows
php src\server.php  # 命令行
```

**配置文件**:
- `config/.env` - 环境变量配置

**要求**:
- PHP 7.4 或更高版本
- 无需额外扩展

---

### 5. C语言版本

**路径**: `05_c_version/`

**特点**:
- 最高性能
- 最小资源占用
- 编译为原生可执行文件

**编译方式**:
```bash
cd 05_c_version
scripts\编译.bat  # Windows (需要MinGW-w64)
```

**启动方式**:
```bash
cd 05_c_version
scripts\启动.bat  # Windows
```

**配置文件**:
- `config/config.h` - 编译时常量配置

**要求**:
- Windows: MinGW-w64 或 MSYS2
- Linux: GCC
- macOS: Xcode Command Line Tools

**编译示例**:
```bash
# Windows
gcc src/main.c -o src/xiaosi-nas-c.exe -lws2_32 -Wall -O2

# Linux
gcc src/main.c -o src/xiaosi-nas-c -lpthread -Wall -O2
```

---

### 6. Shell版本

**路径**: `06_shell_version/`

**特点**:
- Linux/Unix专用
- 依赖Python HTTP服务器
- 轻量级脚本

**启动方式**:
```bash
cd 06_shell_version
chmod +x scripts/启动.sh
./scripts/启动.sh
```

**配置文件**:
- `config/config.sh` - 服务器配置

**要求**:
- Bash shell
- Python 3

---

## ⚙️ 配置管理

### 环境变量 (.env)

适用于: Python, Node.js, PHP

```env
PORT=8080
HOST=0.0.0.0
STORAGE_PATH=../storage
PUBLIC_PATH=../public
JWT_SECRET=your-secret-key
DEFAULT_USER=admin
DEFAULT_PASSWORD=admin123
```

### 配置文件 (config.properties)

适用于: Java

```properties
PORT=8080
HOST=0.0.0.0
STATIC_DIR=../public
STORAGE_DIR=../storage
DEFAULT_USER=admin
DEFAULT_PASSWORD=admin123
```

### 头文件 (config.h)

适用于: C语言

```c
#define PORT 8080
#define ROOT_DIR "../public"
#define STORAGE_DIR "../storage"
```

### Shell配置 (config.sh)

适用于: Shell

```bash
PORT=8080
HOST="0.0.0.0"
PUBLIC_DIR="../public"
STORAGE_DIR="../storage"
```

## 🔐 安全建议

1. **修改默认密码**: 生产环境务必修改 `DEFAULT_PASSWORD`
2. **JWT密钥**: 更换 `JWT_SECRET` 为强随机密钥
3. **端口配置**: 使用非标准端口避免冲突
4. **HTTPS**: 生产环境建议配置HTTPS
5. **防火墙**: 限制访问IP范围

## 🐛 故障排除

### Python版本问题

**问题**: `ModuleNotFoundError: No module named 'flask'`
**解决**: 运行 `pip install -r config/requirements.txt`

### Node.js版本问题

**问题**: `node_modules not found`
**解决**: 运行 `cd config && npm install`

### Java版本问题

**问题**: 编译错误
**解决**: 确保JDK版本 >= 11

### C版本问题

**问题**: `gcc not found`
**解决**: 安装MinGW-w64或MSYS2

### PHP版本问题

**问题**: `PHP not found`
**解决**: 安装PHP或使用XAMPP/WAMP

### Shell版本问题

**问题**: `python not found`
**解决**: 安装Python 3

## 🎯 使用建议

### 开发环境
推荐使用 **Python** 或 **Node.js** 版本：
- 快速迭代
- 丰富的调试工具
- 活跃的生态系统

### 生产环境
根据场景选择：

- **小型项目**: Python或PHP
- **中型项目**: Node.js或Java
- **大型项目**: Java或C
- **高性能需求**: C语言

### 嵌入式/IoT
推荐使用 **C** 或 **Shell** 版本：
- 最小资源占用
- 高度可定制
- 快速启动
