# 小思超级NAS - 多语言版本详细指南

## 📖 概述

本文档详细介绍小思超级NAS的6种语言实现，帮助您选择最适合的版本。

## 📁 统一的文件夹结构

每个语言版本都遵循统一的文件夹结构：

```
语言版本/
├── src/              # 源代码文件
├── config/           # 配置文件
│   ├── .env          # 环境变量配置
│   ├── config.xxx    # 语言特定配置
│   └── package.json  # Node.js依赖配置
└── scripts/          # 脚本文件
    ├── 启动.bat      # Windows启动脚本
    └── 编译.bat      # 编译脚本（仅C版本）
```

## 🚀 各版本详细说明

### 1. Python版本

**路径**: `01_python_version/`

**特点**:
- 易于部署，无需编译
- 自动安装依赖
- 跨平台支持

**启动方式**:
```bash
cd 01_python_version
scripts\启动.bat  # Windows
```

**配置文件**:
- `config/.env` - 服务器端口、JWT密钥等
- `config/requirements.txt` - Python依赖包

**依赖**:
- Flask
- Flask-CORS
- python-dotenv
- bcrypt
- PyJWT

---

### 2. Node.js版本

**路径**: `02_nodejs_version/`

**特点**:

- 丰富的npm生态系统
- 适合Web应用开发

**启动方式**:

scripts\启动.bat  # Windows

**配置文件**:

### 5. C语言版本

**路径**: `05_c_version/`

**特点**:
- 最高性能
- 最小资源占用
- 编译为原生可执行文件

**编译方式**:
```bash
cd 05_c_version
scripts\编译.bat  # Windows (需要MinGW-w64)
```

**启动方式**:
```bash
cd 05_c_version
scripts\启动.bat  # Windows
```

**配置文件**:
- `config/config.h` - 编译时常量配置

**要求**:
- Windows: MinGW-w64 或 MSYS2
- Linux: GCC
- macOS: Xcode Command Line Tools

**编译示例**:
```bash
# Windows
gcc src/main.c -o src/xiaosi-nas-c.exe -lws2_32 -Wall -O2

# Linux
gcc src/main.c -o src/xiaosi-nas-c -lpthread -Wall -O2
```

### 7. Go语言版本 🆕

**路径**: `07_go_version/`

**特点**:
- 高并发性能（Goroutine）
- 编译为单一二进制文件
- 跨平台编译能力
- 内存占用低
- 启动速度快

**启动方式**:
```bash
cd 07_go_version
scripts\启动.bat  # Windows
```

**配置文件**:
- `config/.env` - 环境变量配置
- `config/go.mod` - Go模块依赖

**依赖**:
- gorilla/mux (路由)
- dgrijalva/jwt-go (JWT认证)
- golang.org/x/crypto (bcrypt加密)

**要求**:
- Go 1.21 或更高版本
- 首次运行会自动下载依赖并编译

**编译示例**:
```bash
cd src
go build -o xiaosi-nas.exe main.go
./xiaosi-nas.exe
```

### 8. Rust语言版本 🆕

**路径**: `08_rust_version/`

**特点**:
- **极致性能**（接近C/C++）
- **内存安全**（无空指针、无数据竞争）
- **零成本抽象**
- 无垃圾回收器（GC）
- 类型系统强大

**启动方式**:
```bash
cd 08_rust_version
scripts\启动.bat  # Windows (首次编译较慢，约2-5分钟)
```

**配置文件**:
- `config/.env` - 环境变量配置
- `config/Cargo.toml` - Rust包管理配置

**依赖**:
- actix-web 4 (高性能Web框架)
- jsonwebtoken (JWT认证)
- bcrypt (密码加密)
- serde (序列化)

**要求**:
- Rust 1.75 或更高版本
- Cargo (Rust包管理器)

**性能优势**:
- 并发请求/秒: ~10000+
- 内存占用: <10MB
- 启动时间: <0.1秒
- 二进制体积: <5MB (release模式)

**注意**: 首次编译需要下载依赖，可能需要2-5分钟时间。

## ⚙️ 配置管理

### 环境变量 (.env)

适用于: Python, Node.js, PHP

```env
PORT=8080
HOST=0.0.0.0
STORAGE_PATH=../storage
PUBLIC_PATH=../public
JWT_SECRET=your-secret-key
DEFAULT_USER=admin
DEFAULT_PASSWORD=admin123
```

### 配置文件 (config.properties)

适用于: Java

```properties
PORT=8080
HOST=0.0.0.0
STATIC_DIR=../public
STORAGE_DIR=../storage
DEFAULT_USER=admin
DEFAULT_PASSWORD=admin123
```

### 头文件 (config.h)

适用于: C语言

```c
#define PORT 8080
#define ROOT_DIR "../public"
#define STORAGE_DIR "../storage"
```

### Shell配置 (config.sh)

适用于: Shell

```bash
PORT=8080
HOST="0.0.0.0"
PUBLIC_DIR="../public"
STORAGE_DIR="../storage"
```

## 🔐 安全建议

1. **修改默认密码**: 生产环境务必修改 `DEFAULT_PASSWORD`
2. **JWT密钥**: 更换 `JWT_SECRET` 为强随机密钥
3. **端口配置**: 使用非标准端口避免冲突
4. **HTTPS**: 生产环境建议配置HTTPS
5. **防火墙**: 限制访问IP范围

## 🐛 故障排除

### Python版本问题

**问题**: `ModuleNotFoundError: No module named 'flask'`
**解决**: 运行 `pip install -r config/requirements.txt`

### Node.js版本问题

**问题**: `node_modules not found`
**解决**: 运行 `cd config && npm install`

### Java版本问题

**问题**: 编译错误
**解决**: 确保JDK版本 >= 11

### C版本问题

**问题**: `gcc not found`
**解决**: 安装MinGW-w64或MSYS2

### PHP版本问题

**问题**: `PHP not found`
**解决**: 安装PHP或使用XAMPP/WAMP

### Shell版本问题

**问题**: `python not found`
**解决**: 安装Python 3

## 🎯 使用建议

### 开发环境
推荐使用 **Python** 或 **Node.js** 版本：
- 快速迭代
- 丰富的调试工具
- 活跃的生态系统

### 生产环境
根据场景选择：

- **小型项目**: Python或PHP
- **中型项目**: Node.js或Java
- **大型项目**: Java或C
- **高性能需求**: C语言

### 嵌入式/IoT
推荐使用 **C** 或 **Shell** 版本：
- 最小资源占用
- 高度可定制
- 快速启动
